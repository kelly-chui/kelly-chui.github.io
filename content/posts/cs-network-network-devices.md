## 네트워크 기기의 처리 범위

네트워크 장비는 주로 어떤 계층의 정보를 보고 전달 여부를 결정하는지에 따라 구분한다. 실제 장비는 여러 계층의 기능을 함께 제공할 수 있으므로 아래 구분은 대표적인 처리 기준이다.

- 애플리케이션 계층: L7 스위치
- 인터넷 계층: 라우터, L3 스위치
- 데이터 링크 계층: L2 스위치, 브리지
- 데이터 링크·물리 계층: NIC, 무선 AP
- 물리 계층: 리피터

## 애플리케이션 계층을 처리하는 기기

애플리케이션 계층의 정보를 기반으로 트래픽을 처리하는 장비

### L7스위치(로드 밸런서)

HTTP 헤더, URL 경로, 쿠키처럼 애플리케이션 계층의 정보를 읽어 요청을 적절한 백엔드 서버로 분산하는 장비 또는 소프트웨어

- 서버의 부하를 분산하여 클라이언트로부터 오는 요청들을 뒤쪽의 여러 서버로 나눠 시스템이 처리할 수 있는 트래픽을 증가시킴
- URL, 서버, 캐시, 쿠키를 기반으로 트래픽분산
- 바이러스, 불필요한 외부 데이터 등을 걸러내는 필터링 기능, 트래픽 모니터링
- 장애가 발생한 서버는 헬스 체크를 이용하여 제외함

#### L4 스위치와 L7 스위치 차이

L4 로드 밸런서는 주로 출발지·목적지 IP 주소와 TCP/UDP 포트 같은 정보를 기준으로 트래픽을 분산한다. L7 로드 밸런서는 여기에 HTTP 요청 내용까지 활용할 수 있다.

#### 헬스 체크

L7, L4 모두 헬스 체크를 통해 비정상적인 서버를 판별함. 전송 주기와 재전송 횟수를 설정하여 반복적으로 서버에 요청에 보내는 것으로 수행

#### 로드밸런서를 이용한 서버 이중화

서비스를 안정적으로 운용하기 위해서는 2대 이상의 서버가 필수적임. 로드 밸런서는 2대 이상의 서버를 기반으로 가상 IP를 제공하고 이를 기반으로 안정적인 서비스 제공

## 인터넷 계층을 처리하는 기기

IP 주소를 기반으로 서로 다른 네트워크를 연결하는 장비

### 라우터(router)

- 여러 개의 네트워크를 연결, 분할, 구분시켜주는 역할
- 라우팅 테이블과 라우팅 정책을 기준으로 다음 홉을 선택하여 IP 패킷을 전달한다. 선택된 경로가 항상 물리적으로 가장 짧은 경로인 것은 아니다.

### L3 스위치

L2 스위칭에 더해 IP 라우팅 기능을 제공하는 장비다. 일반적으로 LAN 내부의 VLAN 간 라우팅을 고속으로 처리하는 데 사용하며, 라우터와 기능이 겹치지만 사용 목적과 지원 기능은 제품에 따라 다르다.

## 데이터 링크 계층을 처리하는 기기

같은 LAN에서 MAC 주소를 이용해 데이터를 전달하는 장비

### L2 스위치

장치들의 MAC 주소를 MAC 주소 테이블을 통해 관리하며, 연결된 장치로부터 프레임이 왔을 때 전달할 포트를 결정한다.

IP 주소 기반 라우팅이 아니라 프레임의 목적지 MAC 주소를 보고 스위칭한다. 목적지 MAC 주소가 테이블에 없으면 같은 VLAN의 입력 포트를 제외한 포트로 프레임을 플러딩한다.

### 브리지(bridge)

두 개의 LAN을 상호 접속할 수 있도록 하는 통신망 연결 장치, 포트와 포트 사의 다리 역할, 장치에서 받아온 MAC 주소를 MAC 주소 테이블로 관리

통신망 범위 확장, 서로 다른 LAN 등으로 이루어진 ‘하나의’ 통신망 구축

## 데이터 링크·물리 계층을 처리하는 기기

실제 신호를 송수신하거나 전달하는 장비

### NIC(Network Interface Card, LAN 카드)

호스트를 유선 또는 무선 네트워크에 연결하고 프레임 송수신과 실제 신호 변환을 담당한다. 네트워크 인터페이스에는 일반적으로 MAC 주소가 할당된다.

## 리피터

들어오는 신호를 증폭하여 다른 쪽으로 전달
